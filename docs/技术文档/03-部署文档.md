# 部署文档

## 环境要求

- Node.js 18.x + (建议使用 LTS 版本)
- Python 3.10+ (后端依赖需要较新版本的 Python 支持)
- MySQL 8.0+ (支持 utf8mb4 字符集)
- 微信开发者工具 (用于小程序预览与上传)

## 后端部署 (backend)

### 1. 安装依赖

建议在虚拟环境中运行：

```bash
cd backend
python -m venv venv
# Windows
.\venv\Scripts\activate
# Linux/macOS
source venv/bin/activate

pip install -r requirements.txt
```

### 2. 配置环境变量

复制 `.env.example` 为 `.env`，根据实际情况修改配置：

```env
FLASK_ENV=development
SECRET_KEY=随机生成的密钥

# MySQL 数据库配置
DB_HOST=127.0.0.1
DB_PORT=3306
DB_USER=root
DB_PASSWORD=你的密码
DB_NAME=lbs_secondhand

# 微信小程序配置
WX_APPID=你的小程序AppID
WX_SECRET=你的小程序AppSecret

# AI 发布辅助配置 (豆包/火山方舟，可选)
AI_ENABLED=1
AI_PROVIDER=doubao
ARK_API_KEY=你的API密钥
ARK_MODEL=doubao-1-5-lite-32k-250115
ARK_BASE_URL=https://ark.cn-beijing.volces.com/api/v3
```

### 3. 初始化数据库

```bash
# 1. 在 MySQL 中创建数据库
mysql -u root -p -e "CREATE DATABASE IF NOT EXISTS lbs_secondhand DEFAULT CHARACTER SET utf8mb4;"

# 2. 执行所有迁移脚本（重要：必须执行以建立表结构）
flask db upgrade
```

### 4. 启动服务

系统需要同时运行 HTTP 服务和 WebSocket 服务：

**HTTP API 服务 (默认端口 5002)：**
```bash
python run.py
```

**WebSocket 聊天服务 (默认端口 5001)：**
```bash
python ws_server.py
```

## 后台管理系统部署 (admin-web)

### 1. 安装依赖
```bash
cd admin-web
npm install
```

### 2. 配置环境变量
修改 `src/api/index.ts` 或通过环境变量配置后端接口地址：
- 默认后台登录账号：`admin`
- 默认后台登录密码：`admin123456`

### 3. 运行与编译
```bash
# 本地开发
npm run dev
# 生产环境编译
npm run build
```

## 小程序部署 (miniapp)

### 1. 安装依赖

```bash
cd miniapp
npm install
# 生成 UI 专用本地图标资源（重要，否则图标无法显示）
node scripts/create-ui-icons.js
```

### 2. 配置 API 地址

在开发阶段，由于真机无法直接访问 `127.0.0.1`，需使用电脑局域网 IP。

**修改 `config/index.ts` 或创建 `.env`：**
```env
TARO_APP_API=http://10.10.31.129:5002
TARO_APP_WS=ws://10.10.31.129:5001
```
*(注：请将 IP 地址替换为您电脑的实际局域网 IP)*

### 3. 编译与预览

```bash
# 启动 H5 预览（可选）
npm run dev:h5
# 启动微信小程序预览
npm run dev:weapp
```

### 4. 上传与发布

- 在微信开发者工具中打开 `miniapp/dist` 目录。
- 点击右上角「上传」，填写版本号。
- 登录 [微信公众平台](https://mp.weixin.qq.com/)，将版本提交审核并发布。

## 生产环境注意事项

1. **静态文件**：确保 `backend/uploads` 目录具有读写权限，以便存储上传的商品图片。
2. **跨域配置**：生产环境下需在 `Flask-CORS` 中配置允许的域名。
3. **HTTPS**：微信小程序要求所有 request 和 uploadFile 接口必须使用 HTTPS。
4. **数据库备份**：定期导出 `lbs_secondhand` 数据库 SQL 备份。

## 常见问题

1. **图标显示不出**：请确认是否执行了 `node scripts/create-ui-icons.js`。
2. **登录提示 404/500**：检查后端 `run.py` 是否正常运行，数据库是否已执行 `flask db upgrade`。
3. **图片路径问题**：检查 `fixImageUrl` 函数中配置的 `BASE_URL` 是否与当前环境一致。
