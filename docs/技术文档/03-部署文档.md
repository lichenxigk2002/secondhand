# 部署文档

## 环境要求

- Node.js 18+
- Python 3.8+
- MySQL 8.0+
- 微信开发者工具

## 后端部署

### 1. 安装依赖

```bash
cd backend
pip install -r requirements.txt
```

### 2. 配置环境变量

复制 `.env.example` 为 `.env`，填写：

```env
FLASK_ENV=production
SECRET_KEY=随机生成的密钥

DB_HOST=127.0.0.1
DB_PORT=3306
DB_USER=root
DB_PASSWORD=你的密码
DB_NAME=lbs_secondhand

WX_APPID=你的小程序AppID
WX_SECRET=你的小程序AppSecret
```

### 3. 初始化数据库

```bash
# 创建数据库
mysql -u root -p -e "CREATE DATABASE IF NOT EXISTS lbs_secondhand DEFAULT CHARACTER SET utf8mb4;"

# 执行迁移（推荐）
flask db upgrade
# 或使用 alembic
alembic upgrade head
```

### 4. 启动服务

开发环境：
```bash
python run.py
```

生产环境（推荐使用 gunicorn）：
```bash
pip install gunicorn
gunicorn -w 4 -b 0.0.0.0:5000 "app:create_app()"
```

### 5. 配置 HTTPS

微信小程序要求 request 域名使用 HTTPS。可使用：
- 云服务器 + Nginx 反向代理 + Let's Encrypt
- 或内网穿透（仅开发测试）

## 小程序部署

### 1. 安装依赖

```bash
cd miniapp
npm install
node scripts/create-icons.js
```

### 2. 配置 API 地址

生产环境创建 `.env.production`：
```
TARO_APP_API=https://你的域名.com
```

### 3. 编译

```bash
npm run build:weapp
```

### 4. 上传

- 用微信开发者工具打开 `miniapp/dist`
- 右上角「上传」→ 填写版本号与备注
- 登录微信公众平台 → 版本管理 → 提交审核

### 5. 配置合法域名

微信公众平台 → 开发 → 开发管理 → 开发设置 → 服务器域名：

- request 合法域名：`https://你的API域名`
- uploadFile 合法域名：同上

## 常见问题

1. **登录失败**：检查 WX_APPID、WX_SECRET 是否正确，是否在小程序后台配置
2. **请求超时**：检查服务器防火墙、域名备案、HTTPS 证书
3. **图片不显示**：确认 uploads 目录有读写权限，静态文件路由正确
