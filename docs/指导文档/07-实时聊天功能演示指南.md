# 实时聊天与双端互通演示指南

本指南旨在指导如何利用手机热点构建局域网，实现小程序在“真机”与“电脑模拟器”之间的实时聊天（WebSocket）演示。

## 1. 演示前准备

- **两个微信号**：一个用于手机端（模拟买家/卖家），一个用于电脑端（模拟卖家/买家）。
- **手机热点**：用于提供统一的局域网环境。
- **电脑环境**：已安装并配置好后端 Python 环境及前端 Node.js 环境。

## 2. 步骤一：构建局域网环境

1. **开启热点**：手机开启移动热点。
2. **电脑连接**：电脑连接至该手机热点。
3. **获取电脑局域网 IP**：
   - Windows：按下 `Win + R`，输入 `cmd`，执行 `ipconfig`。
   - 找到“无线局域网适配器 Wi-Fi”下的 **IPv4 地址**（例如：`192.168.43.10`）。

## 3. 步骤二：更新项目配置

由于局域网 IP 发生变化，需要同步更新小程序端的配置：

1. 打开 [miniapp/.env](file:///e:/Project/liuzheng/miniapp/.env) 文件。
2. 将 IP 地址替换为步骤一中获取的新 IP：
   ```env
   TARO_APP_API=http://192.168.43.10:5002
   TARO_APP_WS=ws://192.168.43.10:5001
   ```

## 4. 步骤三：启动后端服务

在电脑端开启两个终端窗口，分别启动 API 和 WebSocket 服务：

**窗口 1 (API 服务)：**
```bash
cd backend
.\venv\Scripts\python.exe run.py
```

**窗口 2 (WebSocket 服务)：**
```bash
cd backend
.\venv\Scripts\python.exe ws_server.py
```

## 5. 步骤四：启动前端并进行真机调试

1. 在电脑端启动小程序编译：
   ```bash
   cd miniapp
   npm run dev:weapp
   ```
2. 打开 **微信开发者工具**：
   - 点击上方工具栏的 **“真机调试”**。
   - 选择“真机调试 1.0”或“2.0”。
   - 使用手机微信扫描生成的二维码。
   - **重要**：手机进入小程序后，点击右上角三个点 -> 开发调试 -> 勾选 **“不校验合法域名”**（如果尚未配置 HTTPS）。

## 6. 步骤五：模拟双端互通

1. **电脑端 (用户 A)**：
   - 在开发者工具模拟器中登录一个微信号。
   - 发布一个闲置商品。
2. **手机端 (用户 B)**：
   - 扫描真机调试二维码进入小程序。
   - 在首页找到刚才用户 A 发布的商品。
   - 点击 **“我想要”**。
3. **即时通讯演示**：
   - 在聊天页面发送消息。
   - 观察电脑端模拟器：消息会通过 WebSocket 实时弹出，无需刷新。
   - 观察手机端：收到回复时同样会实时显示。

## 7. 常见问题排查

- **连不上服务器**：
  - 检查电脑防火墙是否关闭，或是否放行了 `5001` 和 `5002` 端口。
  - 确认手机和电脑确实连接在同一个热点下。
- **消息发送失败**：
  - 确认 `.\venv\Scripts\python.exe ws_server.py` 是否正常运行。
  - 检查手机端是否开启了“不校验合法域名”。
- **IP 地址失效**：
  - 每次重新连接热点，IP 可能会变，请务必通过 `ipconfig` 重新确认并修改 `.env`。
