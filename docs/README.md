# 文档目录说明

当前 `docs` 目录已按用途拆分，便于开发、交付和毕设材料整理。

## 目录结构

```text
docs/
├── README.md
├── 毕设文档/
│   └── 毕业论文-正文初稿.md
├── 需求文档/
│   ├── 01-需求文档.md
│   ├── 06-后台管理系统PRD.md
│   └── 实现功能清单.md
├── 技术文档/
│   ├── 02-技术架构设计.md
│   ├── 03-部署文档.md
│   └── 04-数据库设计.md
└── 指导文档/
    ├── 05-毕业论文写作指南.md
    └── 07-实时聊天功能演示指南.md
```

## 分类说明

### `毕设文档/`

存放毕设正文或阶段性论文材料，适合集中管理老师要求的论文内容。

当前文件：

- `毕业论文-正文初稿.md`

### `需求文档/`

存放产品需求、功能范围、实现清单和后台 PRD 等偏产品与业务规划的文档。

当前文件：

- `01-需求文档.md`
- `06-后台管理系统PRD.md`
- `实现功能清单.md`

### `技术文档/`

存放系统设计、数据库、部署等偏开发实现与交付的文档。

当前文件：

- `02-技术架构设计.md`
- `03-部署文档.md`
- `04-数据库设计.md`

### `指导文档/`

存放论文写作说明、答辩参考、材料编写建议等指导性文档。

当前文件：

- `05-毕业论文写作指南.md`
- `07-实时聊天功能演示指南.md`

## 使用建议

1. 看整体资料时，先从本文件开始。
2. 做功能开发时，优先看 `需求文档/` 和 `技术文档/`。
3. 准备论文和答辩时，优先看 `毕设文档/` 和 `指导文档/`。
4. 后续新增文档时，优先按用途归档，避免重新堆回根目录。
