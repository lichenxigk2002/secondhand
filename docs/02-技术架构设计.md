## 五、核心算法设计

### 5.1 邻近排序算法（地理围栏 + 距离排序）

**输入：** 用户当前位置 `(lat_user, lng_user)`，商品列表（含经纬度）  
**输出：** 按距离由近及远排序的商品列表

**算法思路（ Haversine 公式）：**

```
函数 distance(lat1, lng1, lat2, lng2):
    R = 6371  // 地球半径(km)
    dLat = toRad(lat2 - lat1)
    dLng = toRad(lng2 - lng1)
    a = sin²(dLat/2) + cos(lat1) * cos(lat2) * sin²(dLng/2)
    c = 2 * atan2(√a, √(1-a))
    返回 R * c
```

**地理围栏：** 仅返回距离 ≤ 设定半径（如 3km、5km）内的商品，再按距离排序。

**实现建议：**

- MySQL：使用 `ST_Distance_Sphere(location1, location2)` 计算球面距离，配合空间索引提升性能
- 或在后端（Java/Python）中查询商品后，用 Haversine 公式计算距离并排序

### 5.2 信用评价计算（可选）

可设计简单规则，例如：

- 信用分 = 基础分 + 好评加分 - 差评减分
- 等级：根据分数区间划分为「初级 / 中级 / 高级 / 钻石」等

## 六、数据库设计概要（MySQL）

### 6.1 表结构概要

| 表名 | 说明 | 主要字段 |
|-----|------|----------|
| user | 用户表 | id, openid, nick_name, avatar, phone, campus, credit_score |
| goods | 商品表 | id, title, price, images, category, lat, lng, user_id, status |

| conversation | 会话表（可选） | id, user1_id, user2_id, last_message_id |

## 九、部署与运行环境

### 9.1 生产部署

- **后端**：部署至服务器（腾讯云/阿里云 ECS 或校内服务器）
- **协议**：需配置 HTTPS、域名备案（微信小程序 request 域名要求）
- **数据**：MySQL、Redis（可选）可同机或独立部署
- **存储**：MinIO/阿里云 OSS 存放商品图片
- **小程序**：在 project.config.json 中配置 request 合法域名指向后端 API

### 9.2 开发环境要求

- 操作系统：Windows / macOS
- 网络：稳定互联网（实验室或校园网）
- 工具：微信开发者工具、Visual Studio Code
- 前端：Node.js 18+、Taro 3.x、React 18、TypeScript
- 后端：Python 3.8+、Flask 或 FastAPI
- 数据库：MySQL 8.0+
- 可选：Redis、Postman 等
